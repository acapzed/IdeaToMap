# IdeaToMap

> 팀원 간 이해를 맞추고, 아이디어를 실행 가능한 기획으로 — AI와 함께 대화하며 만드는 프로젝트 플랜

해커톤, 캡스톤, 사이드 프로젝트를 시작하는 팀에게는 공통적인 문제가 있습니다.  
아이디어는 있는데 **팀원마다 머릿속에 그리는 그림이 다르고, 뭘 만들어야 할지 합의가 안 된다는 것.**

기획 문서를 쓰자니 막막하고, 그냥 시작하면 나중에 방향이 엇갈립니다.

IdeaToMap은 AI가 팀 전체에게 같은 질문을 던지며 **공통된 이해를 만들어가는 과정**을 안내합니다.  
"이걸 만들면 어떨까요?" 가 아니라 "이게 맞나요?" — AI가 먼저 추측하고, 팀이 확인하거나 수정하는 방식입니다.  
대화가 끝나면 팀 전체가 동의한 내용이 리포트로 남습니다.

---

## 기획 의도

대부분의 AI 기획 도구는 아이디어를 **넓히는** 방향으로 작동합니다.  
IdeaToMap은 반대입니다 — **팀이 실제로 만들 수 있는 것으로 좁혀나갑니다.**

- **팀원 간 이해 정렬** → 같은 질문에 같이 답하면서 서로 생각이 달랐던 부분을 발견하고 합의
- 자유 입력 대신 **선택지 기반 Q&A** → 애매한 표현 없이 구체적인 결정으로 수렴
- 질문이 미리 정해진 것이 아니라 **이전 답변을 보고 다음 질문을 결정** → 반복 없이 깊어짐
- 기술 스택, 핵심 모듈, 구현 방식까지 구체화 → 개발 시작 전에 "뭘 만들지"가 명확해짐

## 주요 기능

### 1. 아이디어 대화 확인
아이디어를 입력하면 AI가 3~4가지 질문으로 맥락을 파악합니다.  
"멘토링 플랫폼" 하나로는 알 수 없는 것들 — 누구를 위한 건지, 어떤 문제를 푸는 건지 — 을 먼저 정리합니다.

### 2. 동적 Q&A (히스토리 기반 질문 생성)
매 답변 후 AI가 전체 대화 흐름을 보고 **다음에 물어볼 가장 중요한 질문 1개**를 생성합니다.

- 목적 → 팀 구성 → 핵심 기능 → **기술 스택** → **핵심 모듈 상세** → 제약 → 성공 기준 순으로 자연스럽게 진행
- 이미 다룬 내용은 반복하지 않고, 미정으로 넘긴 답변은 다시 파고듦
- AI가 충분히 파악했다고 판단하면 자동으로 리포트 단계로 전환

모든 질문에는:
- **AI 추측 말풍선** — "저는 ~라고 생각해요" 형태로 먼저 가설 제시
- **추천 선택지** — 이 프로젝트 맥락에서 가장 현실적인 옵션 강조
- **"아직 미정이에요"** — 지금 결정하기 어려우면 나중으로 미룰 수 있음

### 3. 실시간 마인드맵
답변할 때마다 마인드맵이 살아납니다.  
처음엔 회색 뼈대 노드로 시작해서, 답변이 쌓일수록 카테고리가 활성화되고 리프 노드가 추가됩니다.

### 4. 종합 리포트
모든 질문이 끝나면 AI가 종합 리포트를 생성합니다.

| 섹션 | 내용 |
|---|---|
| 요약 | 프로젝트 전체를 3~4문장으로 |
| 유사 서비스 분석 | AI가 직접 조사한 유사 앱/사례 + 차별점 |
| 추천 기술 스택 | 팀 수준·기간·기능에 맞는 기술 추천 (라이브러리까지) |
| MVP 핵심 기능 | 우선순위 + 구현 힌트 |
| 리스크 분석 | 심각도별 리스크 + 대응 방안 |
| 팀 & 일정 평가 | 현실성 평가 |
| 지금 당장 해야 할 것 | 구체적 액션 아이템 |

### 5. 심화 탐색
리포트 이후 특정 영역을 더 파고들고 싶으면, 해당 질문을 클릭하면 그 영역에 특화된 심화 질문 3개가 추가됩니다.

### 6. Mermaid 내보내기
마인드맵을 Mermaid 다이어그램 코드로 내보낼 수 있습니다.

## 기술 스택

| 레이어 | 기술 |
|---|---|
| 프론트엔드 | React + Vite, Tailwind CSS, Zustand |
| 마인드맵 | @xyflow/react (react-flow) + dagre 자동 레이아웃 |
| 백엔드 | Node.js + Express |
| AI | Groq API (Meta Llama 3.3 70B) |
| 인프라 | Docker Compose |

AI 호출은 모두 백엔드에서만 처리합니다.  
`AI_BASE_URL` 환경변수 하나로 Groq → 자체 GPU 서버로 마이그레이션이 가능합니다.

## 실행 방법

### 사전 준비
- Docker Desktop
- [Groq API 키](https://console.groq.com) (무료 티어 사용 가능)

### 시작

```bash
# 1. 저장소 클론
git clone https://github.com/acapzed/IdeaToMap.git
cd IdeaToMap

# 2. 환경변수 설정
cp .env.example .env
# .env 파일에 GROQ_API_KEY 입력

# 3. 실행
docker compose up --build
```

브라우저에서 `http://localhost:5173` 접속

## 앱 흐름

```
아이디어 입력
    ↓ (유효성 검사)
아이디어 대화 확인 (clarify Q&A)
    ↓
동적 Q&A (히스토리 기반, 9~12회)
    ├── 답변마다 마인드맵 실시간 업데이트
    └── AI가 기술스택·핵심모듈까지 파고듦
    ↓
종합 리포트 생성
    ├── 리포트 탭: 분석 결과 전체
    └── 마인드맵 탭: 인터랙티브 그래프
```

## API 엔드포인트

```
POST /api/sessions                    세션 생성 (아이디어 유효성 검사 포함)
POST /api/sessions/:id/confirm        아이디어 파악용 질문 생성
POST /api/sessions/:id/confirm/done   대화 답변 → 요약 생성
POST /api/sessions/:id/questions      첫 번째 질문 생성
POST /api/sessions/:id/answers        답변 제출 → 다음 질문 동적 생성
POST /api/sessions/:id/mindmap/enrich 종합 리포트 생성
POST /api/sessions/:id/explore        특정 영역 심화 질문 생성
GET  /api/sessions/:id                세션 전체 상태 조회
```
